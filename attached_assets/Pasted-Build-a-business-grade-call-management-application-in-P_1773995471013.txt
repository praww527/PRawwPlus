Build a business-grade call management application in Python (using Flask or FastAPI) on Replit, integrated with Telnyx for calling and MongoDB for data storage. Key business logic includes:
1.	Subscription Model: Each user subscribes to a plan costing R100 per month. After payment, they receive R20 of call credit each month. The balance resets only after a successful monthly payment, verified by a payment processor like Stripe.
2.	Payment Integration: Use Stripe to handle monthly recurring payments. A successful payment triggers a webhook, which updates the user’s balance with a fresh R20 credit. Only when payment is confirmed does the balance reset—no automatic date-based resets.
3.	Call Tracking: Use Telnyx API to make calls, and log each call (caller, recipient, duration, cost) in MongoDB. Each user record stores their total call usage. If a user’s credit is exhausted, block further calls until the next payment cycle.
4.	User Management: Each user has a secure login, and their subscription status is tied to their payment. Store a “last payment date” field, so you know when the next credit reset should occur.
5.	Frontend Design: Build a modern UI with a frosted glass effect, round edges, and a minimal layout using Tailwind CSS. Show the user’s current balance, subscription status, and allow them to see call history and buy additional credits if needed.
6.	Environment Security: Store all API keys (Telnyx, Stripe) and sensitive details in environment variables. Ensure all API calls are securely made over HTTPS.
7.	Admin Dashboard: Provide an admin view to monitor user subscriptions, payments, call usage, and balances.
Ensure the logic only resets a user’s free credit after a confirmed monthly payment. If a payment fails or is pending, the balance stays until the next cycle. Once built, the app should scale, securely track usage, and allow users to manage their subscription and call credits.
